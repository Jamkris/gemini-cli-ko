# CLI 제거 (Uninstalling the CLI)

제거 방법은 CLI를 실행한 방식에 따라 다릅니다. npx 또는 npm 전역 설치에 대한 지침을 따르세요.

## 방법 1: npx 사용 시 (Method 1: Using npx)

npx는 영구 설치 없이 임시 캐시에서 패키지를 실행합니다. CLI를 "제거"하려면 이 캐시를 지워야 하며, 이렇게 하면 gemini-cli와 npx로 이전에 실행된 다른 패키지들이 제거됩니다.

npx 캐시는 기본 npm 캐시 폴더 내에 있는 `_npx`라는 디렉터리입니다. `npm config get cache`를 실행하여 npm 캐시 경로를 찾을 수 있습니다.

**macOS / Linux**

```bash
# The path is typically ~/.npm/_npx
rm -rf "$(npm config get cache)/_npx"
```

**Windows**

_명령 프롬프트_

```cmd
:: The path is typically %LocalAppData%\npm-cache\_npx
rmdir /s /q "%LocalAppData%\npm-cache\_npx"
```

_PowerShell_

```powershell
# The path is typically $env:LocalAppData\npm-cache\_npx
Remove-Item -Path (Join-Path $env:LocalAppData "npm-cache\_npx") -Recurse -Force
```

## 방법 2: npm 사용 시 (전역 설치)

CLI를 전역으로 설치한 경우(예: `npm install -g @google/gemini-cli`), `-g` 플래그와 함께 `npm uninstall` 명령을 사용하여 제거하세요.

```bash
npm uninstall -g @google/gemini-cli
```

이 명령은 시스템에서 패키지를 완전히 제거합니다.

## 방법 3: Homebrew

Homebrew를 사용하여 CLI를 전역으로 설치한 경우(예: `brew install gemini-cli`), `brew uninstall` 명령을 사용하여 제거하세요.

```bash
brew uninstall gemini-cli
```

## 방법 4: MacPorts

MacPorts를 사용하여 CLI를 전역으로 설치한 경우(예: `sudo port install gemini-cli`), `port uninstall` 명령을 사용하여 제거하세요.

```bash
sudo port uninstall gemini-cli
```
