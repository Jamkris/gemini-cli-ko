# Gemini CLI의 Gemini 3 Pro 및 Gemini 3 Flash

Gemini 3 Pro 및 Gemini 3 Flash를 이제 모든 사용자가 Gemini CLI에서 사용할 수 있습니다!

## Gemini CLI에서 Gemini 3 시작하기

Gemini CLI를 최신 버전으로 업그레이드하여 시작하세요.

```bash
npm install -g @google/gemini-cli@latest
```

버전이 0.21.1 이상인지 확인한 후:

1. Gemini CLI에서 `/settings` 명령을 사용합니다.
2. **Preview Features**를 `true`로 전환합니다.
3. `/model`을 실행하고 **Auto (Gemini 3)**를 선택합니다.

자세한 내용은 [Gemini CLI 모델 선택](/docs/cli/model)을 참조하세요.

### 사용 한도 및 대체(fallback)

Gemini CLI는 Gemini 3 Pro 일일 사용 한도에 도달하면 알려줍니다.
한도에 도달하면 Gemini 2.5 Pro로 전환하거나, 더 높은 한도로 업그레이드하거나, 중지할 수 있는 옵션이 제공됩니다. 또한 사용 한도가 재설정되고 Gemini 3 Pro를 다시 사용할 수 있는 시점도 알려줍니다.

마찬가지로 Gemini 2.5 Pro의 일일 사용 한도에 도달하면 Gemini 2.5 Flash로 대체를 묻는 메시지가 표시됩니다.

### 용량 오류

Gemini 3 Pro 모델이 과부하되는 경우가 있을 수 있습니다. 이런 경우,
Gemini CLI는 Gemini 3 Pro를 계속 시도할지 아니면 Gemini 2.5 Pro로 대체할지 결정하도록 요청합니다.

> **참고:** **Keep trying** 옵션은 시스템이 사용 중일 때 각 재시도 사이에 대기 시간을 늘리는 지수 백오프를 사용합니다. 재시도가 즉시 발생하지 않는 경우 요청이 처리될 때까지 몇 분 정도 기다리세요.

### 모델 선택 및 라우팅 유형

Gemini CLI를 사용할 때 요청이 모델 간에 라우팅되는 방식을 제어하고 싶을 수 있습니다. 기본적으로 Gemini CLI는 **Auto** 라우팅을 사용합니다.

Gemini 3 Pro를 사용할 때 사용 한도를 관리하기 위해 Auto 라우팅 또는 Pro 라우팅을 사용할 수 있습니다.

- **Auto 라우팅:** Auto 라우팅은 먼저 프롬프트에 복잡한 작업이 포함되는지 아니면 간단한 작업이 포함되는지 결정합니다. 간단한 프롬프트의 경우 자동으로 Gemini 2.5 Flash를 사용합니다. 복잡한 프롬프트의 경우 Gemini 3 Pro가 활성화되어 있으면 Gemini 3 Pro를 사용하고, 그렇지 않으면 Gemini 2.5 Pro를 사용합니다.
- **Pro 라우팅:** 작업이 가장 유능한 모델에서 처리되도록 하려면 `/model`을 사용하고 **Pro**를 선택하세요. Gemini CLI는 활성화된 경우 Gemini 3 Pro를 포함하여 사용 가능한 가장 유능한 모델을 우선시합니다.

모델 선택 및 라우팅에 대해 자세히 알아보려면 [Gemini CLI 모델 선택](/docs/cli/model)을 참조하세요.

## Gemini Code Assist에서 Gemini CLI로 Gemini 3 활성화하기

Gemini Code Assist Standard 또는 Gemini Code Assist Enterprise를 사용하는 경우, Gemini CLI에서 Gemini 3 Pro를 활성화하려면 출시 채널을 구성해야 합니다.
Gemini 3 Pro를 사용하려면 관리자 활성화와 사용자 활성화라는 두 단계를 거쳐야 합니다.

이러한 설정에 대해 자세히 알아보려면 [Gemini Code Assist 출시 채널 구성](https://developers.google.com/gemini-code-assist/docs/configure-release-channels)을 참조하세요.

### 관리자 지침

**Google Cloud Settings Admin** 권한이 있는 관리자는 다음 지침을 따라야 합니다.

- Gemini CLI for Code Assist와 함께 사용 중인 Google Cloud 프로젝트로 이동합니다.
- **Admin for Gemini** > **Settings**로 이동합니다.
- **Release channels for Gemini Code Assist in local IDEs**에서 **Preview**를 선택합니다.
- **Save changes**를 클릭합니다.

### 사용자 지침

관리자가 **Preview**를 활성화한 후 2~3분 정도 기다렸다가 다음을 수행합니다.

- Gemini CLI를 엽니다.
- `/settings` 명령을 사용합니다.
- **Preview Features**를 `true`로 설정합니다.

Gemini CLI를 다시 시작하면 Gemini 3에 액세스할 수 있습니다.
